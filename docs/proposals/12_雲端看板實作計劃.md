# Plan C：雲端看板 + 手機監控 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 將 Pi5 咖啡豆計數結果推送到雲端，讓使用者可用手機在任何地方查看今日總數、批次列表、產線 throughput 曲線、平均輸送帶速度與即時累計數。

**Architecture:** Pi5 只負責本地即時計數與可靠上傳；Supabase 負責 Postgres、REST/RPC API、Realtime、RLS；手機端是一個無框架 PWA，部署在 Cloudflare Pages，登入後直接讀 Supabase。v1 先做「批次完成後上傳 summary」，再加「每 10 秒 live sample」，避免雲端功能影響 7 吋 pygame 主畫面與 Hailo 計數穩定性。

**Tech Stack:** Raspberry Pi 5, Python `requests`, local SQLite retry queue, Supabase Postgres + RPC + Realtime + RLS, Supabase Auth, Cloudflare Pages, Vanilla HTML/CSS/JS, Canvas chart.

## 1. 架構選型

### 推薦方案：Supabase + Cloudflare Pages

```
Pi5 preview_display.py
    │
    ├─ batch finalized
    │     └─ spectral_capture/cloud_uploader.py
    │          └─ POST https://<project_ref>.supabase.co/rest/v1/rpc/ingest_batch
    │
    └─ optional live thread, every 10s
          └─ POST https://<project_ref>.supabase.co/rest/v1/rpc/ingest_sample

Supabase
    ├─ Postgres tables: devices, batches, throughput_samples, current_counts
    ├─ RPC functions: ingest_batch, ingest_sample
    ├─ Realtime publication for current_counts and throughput_samples
    ├─ RLS: dashboard users can read, anonymous users cannot read tables
    └─ device token hash table: only valid Pi5 token can write through RPC

Phone browser
    └─ Cloudflare Pages static PWA
         ├─ Supabase Auth login
         ├─ REST reads for initial dashboard
         ├─ Realtime subscription for current_counts
         └─ polling fallback every 15s
```

### 選型比較

| 選項 | 免費額度與能力 | 優點 | 缺點 | 判斷 |
|---|---:|---|---|---|
| Supabase | Free plan: 500 MB database、5 GB egress、50,000 MAU、500,000 Edge Function invocations、2 million Realtime messages、200 Realtime peak connections；官方文件也說 Data REST API 會自動從 schema 產生，URL 是 `https://<project_ref>.supabase.co/rest/v1/`。 | Postgres schema 清楚；REST/RPC 不必自架 API；Realtime 適合手機 live number；RLS 和 Auth 可直接做簡單安全邊界；solo maker 低維運。 | 免費專案有空間與流量上限；Postgres Changes 需要開 publication；若長期高頻 live sample 會累積資料量。 | **採用**。最少維運、最貼近需求。 |
| Firebase Firestore | 官方 free tier: 1 GiB stored data、50,000 reads/day、20,000 writes/day、20,000 deletes/day、10 GiB outbound/month。 | Realtime 與手機 app 生態強；前端 SDK 成熟。 | NoSQL 不適合 batch + time-series SQL 查詢；RLS/SQL view 不如 Supabase 直覺；長期分析會較繞。 | 可行但不首選。 |
| Cloudflare Worker + D1/KV | D1 Free: 5 million rows read/day、100,000 rows written/day、5 GB storage；KV Free: 100,000 reads/day、1,000 writes/day、1 GB storage。 | Pages 靜態部署非常簡單；D1 免費寫入額度高；全球邊緣快。 | 需要自己寫 Worker API 與 auth；Realtime 不內建；KV 每日 write limit 不適合 live sample。 | 適合 v2 API 代理，不適合 v1 最短路徑。 |
| 自架 FastAPI + Postgres VPS | 取決於 VPS，通常每月 USD 4-6 起。 | 完全可控；可用現有 FastAPI 經驗。 | 不是免費；需要維運、TLS、備份、安全更新；solo maker 今天要上線太重。 | 不採用 v1。 |

### Python uploader module

```python
# spectral_capture/cloud_uploader.py
from __future__ import annotations

import json
import os
import sqlite3
import time
from dataclasses import dataclass
from datetime import datetime, timezone
from pathlib import Path
from typing import Any, Dict, Optional

import requests


ENV_PATH = Path("/home/kyle/.huyes_cloud.env")
DEFAULT_QUEUE_DB = Path("/home/kyle/KyleClaude/spectral_capture/data/cloud_upload_queue.db")


@dataclass(frozen=True)
class CloudConfig:
    supabase_url: str
    supabase_anon_key: str
    device_id: str
    device_token: str
    queue_db: Path
    timeout_s: float = 5.0


def _parse_env(path: Path = ENV_PATH) -> dict[str, str]:
    values: dict[str, str] = {}
    if not path.exists():
        return values
    for raw_line in path.read_text().splitlines():
        line = raw_line.strip()
        if not line or line.startswith("#") or "=" not in line:
            continue
        key, value = line.split("=", 1)
        values[key.strip()] = value.strip().strip('"').strip("'")
    return values


def load_config(path: Path = ENV_PATH) -> CloudConfig:
    file_values = _parse_env(path)
    merged = {**file_values, **{k: v for k, v in os.environ.items() if k.startswith("HUYES_") or k.startswith("SUPABASE_")}}
    required = ["SUPABASE_URL", "SUPABASE_ANON_KEY", "HUYES_DEVICE_ID", "HUYES_DEVICE_TOKEN"]
    missing = [key for key in required if not merged.get(key)]
    if missing:
        raise RuntimeError(f"Missing cloud config: {', '.join(missing)}")
    return CloudConfig(
        supabase_url=merged["SUPABASE_URL"].rstrip("/"),
        supabase_anon_key=merged["SUPABASE_ANON_KEY"],
        device_id=merged["HUYES_DEVICE_ID"],
        device_token=merged["HUYES_DEVICE_TOKEN"],
        queue_db=Path(merged.get("HUYES_QUEUE_DB", str(DEFAULT_QUEUE_DB))),
        timeout_s=float(merged.get("HUYES_UPLOAD_TIMEOUT_S", "5")),
    )


def utc_now_iso() -> str:
    return datetime.now(timezone.utc).isoformat()


def init_queue(db_path: Path) -> None:
    db_path.parent.mkdir(parents=True, exist_ok=True)
    with sqlite3.connect(str(db_path)) as conn:
        conn.execute(
            """
            create table if not exists upload_queue (
              id integer primary key autoincrement,
              kind text not null,
              payload_json text not null,
              attempts integer not null default 0,
              next_attempt_at real not null default 0,
              last_error text,
              created_at real not null default (unixepoch()),
              updated_at real not null default (unixepoch())
            )
            """
        )
        conn.execute("create index if not exists idx_upload_queue_next on upload_queue(next_attempt_at, id)")


def _headers(config: CloudConfig) -> dict[str, str]:
    return {
        "apikey": config.supabase_anon_key,
        "Authorization": f"Bearer {config.supabase_anon_key}",
        "Content-Type": "application/json",
    }


def post_rpc(config: CloudConfig, function_name: str, payload: Dict[str, Any]) -> None:
    url = f"{config.supabase_url}/rest/v1/rpc/{function_name}"
    body = {
        "p_device_id": config.device_id,
        "p_device_token": config.device_token,
        "p_payload": payload,
    }
    resp = requests.post(url, headers=_headers(config), json=body, timeout=config.timeout_s)
    if resp.status_code >= 300:
        raise RuntimeError(f"Supabase RPC {function_name} failed: {resp.status_code} {resp.text[:500]}")


def queue_upload(config: CloudConfig, kind: str, payload: Dict[str, Any], error: str) -> None:
    init_queue(config.queue_db)
    with sqlite3.connect(str(config.queue_db)) as conn:
        conn.execute(
            """
            insert into upload_queue(kind, payload_json, attempts, next_attempt_at, last_error, updated_at)
            values (?, ?, 0, ?, ?, unixepoch())
            """,
            (kind, json.dumps(payload, ensure_ascii=False, separators=(",", ":")), time.time() + 30, error[:1000]),
        )


def summarize_batch_json(batch_json_path: Path, extra: Optional[dict[str, Any]] = None) -> dict[str, Any]:
    raw = json.loads(batch_json_path.read_text())
    total = int(raw.get("total_crossed", raw.get("total_beans", 0)) or 0)
    duration_s = int(raw.get("duration_s") or 0)
    avg_throughput = round(total / duration_s, 3) if duration_s > 0 else 0
    payload = {
        "batch_id": str(raw["batch_id"]),
        "started_at": raw.get("started_at", ""),
        "ended_at": raw.get("ended_at") or utc_now_iso(),
        "duration_s": duration_s,
        "total_beans": int(raw.get("total_beans", total) or 0),
        "total_crossed": total,
        "avg_throughput_beans_s": raw.get("avg_throughput_beans_s", avg_throughput),
        "avg_belt_speed_px_s": raw.get("avg_belt_speed_px_s", 0),
        "source_json_path": str(batch_json_path),
        "frame_count": len(raw.get("frames", [])),
    }
    if extra:
        payload.update(extra)
    return payload


def upload_batch_json(batch_json_path: Path, config: Optional[CloudConfig] = None) -> bool:
    config = config or load_config()
    payload = summarize_batch_json(batch_json_path)
    try:
        post_rpc(config, "ingest_batch", payload)
        drain_queue_once(config, limit=5)
        return True
    except Exception as exc:
        queue_upload(config, "batch", payload, str(exc))
        return False


def upload_live_sample(sample: Dict[str, Any], config: Optional[CloudConfig] = None) -> bool:
    config = config or load_config()
    try:
        post_rpc(config, "ingest_sample", sample)
        return True
    except Exception as exc:
        queue_upload(config, "sample", sample, str(exc))
        return False


def drain_queue_once(config: Optional[CloudConfig] = None, limit: int = 20) -> int:
    config = config or load_config()
    init_queue(config.queue_db)
    now = time.time()
    sent = 0
    with sqlite3.connect(str(config.queue_db)) as conn:
        rows = conn.execute(
            """
            select id, kind, payload_json, attempts
            from upload_queue
            where next_attempt_at <= ?
            order by id
            limit ?
            """,
            (now, limit),
        ).fetchall()
        for row_id, kind, payload_json, attempts in rows:
            payload = json.loads(payload_json)
            function_name = "ingest_batch" if kind == "batch" else "ingest_sample"
            try:
                post_rpc(config, function_name, payload)
                conn.execute("delete from upload_queue where id = ?", (row_id,))
                sent += 1
            except Exception as exc:
                delay_s = min(3600, 30 * (2 ** min(int(attempts), 6)))
                conn.execute(
                    """
                    update upload_queue
                    set attempts = attempts + 1,
                        next_attempt_at = ?,
                        last_error = ?,
                        updated_at = unixepoch()
                    where id = ?
                    """,
                    (time.time() + delay_s, str(exc)[:1000], row_id),
                )
                break
    return sent
```

def start_cloud_live_worker(cloud_state, cloud_lock, interval_s=10):
    if upload_live_sample is None:
        return

    def _worker():
        last_count = 0
        while True:
            time.sleep(interval_s)
            with cloud_lock:
                snapshot = dict(cloud_state)
            current = int(snapshot.get("cumulative_count", 0) or 0)
            count_in_window = max(0, current - last_count)
            last_count = current
            sample = {
                "batch_id": snapshot.get("batch_id", "screen-session"),
                "sampled_at": iso_utc_now(),
                "window_s": interval_s,
                "count_in_window": count_in_window,
                "cumulative_count": current,
                "total_crossed": int(snapshot.get("total_crossed", current) or 0),
                "throughput_beans_s": round(count_in_window / interval_s, 3),
                "belt_speed_px_s": float(snapshot.get("belt_speed_px_s", 0) or 0),
                "live_tracks": int(snapshot.get("live_tracks", 0) or 0),
                "fps": float(snapshot.get("fps", 0) or 0),
                "status": snapshot.get("status", "running"),
            }
            upload_live_sample(sample)
            if drain_queue_once is not None:
                drain_queue_once(limit=10)

    threading.Thread(target=_worker, daemon=True).start()

## 6. Security Plan

### Key rules

- 不要把 `service_role` key 放進 repo、手機 JS、Cloudflare Pages、或任何前端。
- v1 Pi5 使用 `SUPABASE_ANON_KEY + HUYES_DEVICE_TOKEN` 呼叫 RPC；token 只存在 `/home/kyle/.huyes_cloud.env`。
- Direct table insert/update/delete 對 `anon` 與 `authenticated` 全部 revoke；只有 RPC 可以寫。
- RPC 會用 `device_tokens.token_hash` 驗證 device token，驗證成功才 insert/upsert。
- `create_device_secret()` 是管理用函式，只在 Supabase SQL Editor 由專案管理者執行；migration 會對 `public`, `anon`, `authenticated` 明確 `revoke execute`。
- Dashboard 必須 Supabase Auth 登入；RLS 只允許 `dashboard_users` 內的 user 讀資料。
- `service_role` 只作為 Supabase Dashboard、migration、資料修復使用，不放到 Pi5 也可以完成 v1。

## 8. Cost and Limits

### Expected v1 usage

Assume one Pi5, one phone, 8 hours/day:

- Batch uploads: usually less than 100 rows/day。
- Live samples at 10 second interval: `6/min * 60 * 8 = 2,880 rows/day`。
- 30 days live samples: `86,400 rows/month`。
- Estimated row storage: well under 500 MB/month if only numeric samples are stored。
- Realtime messages: approximately one `current_counts` update and one `throughput_samples` insert per interval, under 200,000 messages/month for one phone, far below 2 million/month。
- Realtime peak connections: one phone, far below 200 concurrent。
- Egress: dashboard reads small JSON rows, far below 5 GB/month。

### Free tier fit

Supabase Free is sufficient for solo maker v1:

- 500 MB database: enough for batch summaries and one month of 10 second samples。
- 5 GB egress: enough for small JSON dashboard traffic。
- 2 million Realtime messages: enough for one phone monitoring one device。
- 200 Realtime peak connections: enough for solo maker use。
- 50,000 MAU: far above one dashboard user。

When data grows, first reduce storage by retaining raw `throughput_samples` for 30-90 days and keeping daily aggregates forever.

## 9. Operational Notes

- Keep local batch JSON as source of truth. Cloud is a monitor, not the only record。
- Upload failures should never block or crash `preview_display.py`。
- Queue retry should be best-effort and bounded by exponential backoff。
- Live samples are for dashboard smoothness; batch upload is the durable accounting record。
- If phone and Pi5 are on the same Tailscale network, the existing FastAPI cockpit still works for control. Plan C adds off-site monitoring through cloud, not remote control。
- If Supabase is down, production continues locally and uploads catch up when network returns。
