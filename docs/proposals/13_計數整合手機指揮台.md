> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 Pi5 即時咖啡豆計數整合進既有手機「採集指揮台」，讓同一個 `control_server` 網頁同時控制採集與顯示 live counting。

**Tech Stack:** Raspberry Pi 5, Python 3, pygame, Hailo detector, `BeanTracker`, `/dev/shm` JSON IPC, FastAPI, Vanilla HTML/CSS/JS, Canvas chart, Tailscale local URL `http://100.65.98.76:8765`。

## 1. 架構與狀態共享

### 推薦 IPC

使用 `/dev/shm/count_status.json` 作為唯一 live count IPC：

- Writer: `spectral_capture/preview_display.py`
- Reader: `spectral_capture/control_server.py`
- Update rate: 每 `0.5s` 最多一次，或計數狀態有變化時也仍受 `0.5s` 節流限制
- Write pattern: 寫入 `count_status.json.tmp` 後 `os.replace(tmp, live)`，避免 FastAPI 讀到半截 JSON
- Payload size: 幾百 bytes，不包含 frame image，不包含所有 bbox history

### Flow

```text
Hailo detector + BeanTracker
inside preview_display.py
        |
        | every ~0.5s atomic JSON write
        v
/dev/shm/count_status.json
        |
        | GET /api/count reads file
        v
control_server.py / FastAPI :8765
        |
        | phone polling every 1s
        v
ui/index.html on phone
採集控制 + 即時計數 in one cockpit
```

### `/dev/shm/count_status.json` contract

```json
{
  "schema_version": 1,
  "running": true,
  "paused": false,
  "batch_id": "20260613-001",
  "total_crossed": 403,
  "new_crossings": 1,
  "live_tracks": 7,
  "detected_count": 9,
  "fps": 5.6,
  "frame_id": 12345,
  "throughput_per_min": 86.4,
  "throughput_window_s": 60,
  "belt_speed_px_s": 672.0,
  "updated_at": 1781280000.123
}
```

欄位定義：

| Field | Type | Owner | 說明 |
|---|---:|---|---|
| `schema_version` | int | `preview_display.py` | 固定 `1`，讓手機端與雲端後續可辨識格式 |
| `running` | bool | `preview_display.py` | `capture_meta.json` 是否存在 |
| `paused` | bool | `preview_display.py` | pygame 暫停按鈕狀態 |
| `batch_id` | string | `preview_display.py` | 來自 `/dev/shm/capture_meta.json`，沒有時為 `""` |
| `total_crossed` | int | `BeanTracker` | 本 session 累計通過數，手機大數字使用這個，不使用 DB 總豆數 |
| `new_crossings` | int | `BeanTracker` | 最近一次 `tracker.update()` 產生的新 crossing |
| `live_tracks` | int | `BeanTracker` | 畫面中正在追蹤的 tracks |
| `detected_count` | int | detector JSON | 當前 frame detector box 數 |
| `fps` | float | `preview_status.json` | 相機 preview fps |
| `frame_id` | int | `preview_status.json` | 最新 frame id |
| `throughput_per_min` | float | derived | 最近 60 秒 crossing rate，單位 beans/min |
| `throughput_window_s` | int | constant | v1 固定 60 |
| `belt_speed_px_s` | float | derived | `median(tracker.vh) * fps`，單位 px/s |
| `updated_at` | float | writer clock | Unix epoch seconds，用於 stale 判斷 |

FastAPI 的 `/api/count` 應額外包裝：

```json
{
  "ok": true,
  "stale": false,
  "age_s": 0.4,
  "data": { "...": "count_status fields" }
}
```

當檔案不存在、JSON 壞掉、或超過 `3s` 未更新時，不回 500；回傳 `ok: false` 或 `stale: true`，讓手機顯示「等待計數」但控制功能不受影響。

### 3.2 新增 atomic JSON helper 與 publisher

放在 `read_json()` 後方，或放在 `save_batch_json()` 前方。這個 class 不碰 pygame rendering，也不碰 detector，只拿 main loop 已經算好的數值。

```python
def write_json_atomic(path: Path, payload: dict) -> None:
    tmp = path.with_suffix(path.suffix + '.tmp')
    tmp.write_text(json.dumps(payload, ensure_ascii=False, separators=(',', ':')))
    os.replace(tmp, path)


class CountStatusPublisher:
    def __init__(
        self,
        path: Path = COUNT_STATUS_JSON,
        min_interval_s: float = COUNT_STATUS_WRITE_INTERVAL_S,
        window_s: float = THROUGHPUT_WINDOW_S,
    ):
        self.path = path
        self.min_interval_s = min_interval_s
        self.window_s = window_s
        self.history = deque()
        self.last_write_t = 0.0

    def reset(self) -> None:
        self.history.clear()
        self.last_write_t = 0.0

    def record_count(self, now: float, total_crossed: int) -> None:
        self.history.append((now, int(total_crossed)))
        cutoff = now - self.window_s
        while len(self.history) > 1 and self.history[0][0] < cutoff:
            self.history.popleft()

    def throughput_per_min(self) -> float:
        if len(self.history) < 2:
            return 0.0
        t0, c0 = self.history[0]
        t1, c1 = self.history[-1]
        dt = max(0.001, t1 - t0)
        return max(0.0, (c1 - c0) / dt * 60.0)

    def belt_speed_px_s(self, tracker: BeanTracker, fps: float) -> float:
        if not tracker.vh:
            return 0.0
        return float(np.median(np.asarray(tracker.vh, dtype=np.float32)) * max(0.0, fps))

    def maybe_write(
        self,
        *,
        running: bool,
        paused: bool,
        batch_id: str,
        total_crossed: int,
        new_crossings: int,
        live_tracks: int,
        detected_count: int,
        fps: float,
        frame_id: int,
        tracker: BeanTracker,
    ) -> None:
        now = time.time()
        self.record_count(now, total_crossed)
        if now - self.last_write_t < self.min_interval_s:
            return

        payload = {
            "schema_version": 1,
            "running": bool(running),
            "paused": bool(paused),
            "batch_id": str(batch_id or ""),
            "total_crossed": int(total_crossed),
            "new_crossings": int(new_crossings),
            "live_tracks": int(live_tracks),
            "detected_count": int(detected_count),
            "fps": round(float(fps or 0.0), 3),
            "frame_id": int(frame_id or 0),
            "throughput_per_min": round(self.throughput_per_min(), 2),
            "throughput_window_s": int(self.window_s),
            "belt_speed_px_s": round(self.belt_speed_px_s(tracker, float(fps or 0.0)), 2),
            "updated_at": now,
        }
        try:
            write_json_atomic(self.path, payload)
            self.last_write_t = now
        except Exception as exc:
            print(f"[display] count status write failed: {exc}")
```

```python
@app.get("/api/count")
def count_status():
    data = _read_json_file(COUNT_STATUS_PATH)
    if not data:
        return {
            "ok": False,
            "stale": True,
            "age_s": None,
            "data": {
                "schema_version": 1,
                "running": _is_running(),
                "paused": False,
                "batch_id": "",
                "total_crossed": 0,
                "new_crossings": 0,
                "live_tracks": 0,
                "detected_count": 0,
                "fps": 0.0,
                "frame_id": 0,
                "throughput_per_min": 0.0,
                "throughput_window_s": 60,
                "belt_speed_px_s": 0.0,
                "updated_at": None,
            },
        }

    now = time.time()
    updated_at = data.get("updated_at")
    age_s = None
    stale = True
    if isinstance(updated_at, (int, float)):
        age_s = max(0.0, now - float(updated_at))
        stale = age_s > COUNT_STALE_AFTER_S

    return {
        "ok": True,
        "stale": stale,
        "age_s": round(age_s, 2) if age_s is not None else None,
        "data": data,
    }
```

### 4.4 新增 `GET /api/batches`

目的：手機計數 tab 顯示最近批次，不查 SQLite，不解析所有 frame，只讀 batch JSON summary。

```python
@app.get("/api/batches")
def batches(limit: int = 10):
    limit = max(1, min(int(limit or 10), 50))
    if not BATCH_DIR.exists():
        return {"batches": []}

    rows = []
    for path in sorted(BATCH_DIR.glob("batch_*.json"), reverse=True)[:limit]:
        payload = _read_json_file(path) or {}
        stat = path.stat()
        frames = payload.get("frames") or []
        rows.append({
            "file": path.name,
            "batch_id": payload.get("batch_id") or path.stem.replace("batch_", ""),
            "total_beans": int(payload.get("total_beans") or 0),
            "total_crossed": int(payload.get("total_crossed") or payload.get("total_beans") or 0),
            "frames": len(frames),
            "saved_at": datetime.datetime.fromtimestamp(stat.st_mtime).isoformat(timespec="seconds"),
        })
    return {"batches": rows}
```

### 4.5 v2 optional: `GET /api/preview.jpg`

這個 endpoint 不進 v1 must-have。只有計數 tab 開啟且使用者打開預覽時才 2-5 fps polling，避免 Pi5 CPU 被 PPM to JPEG encode 拖慢。

FastAPI route：

```python
from fastapi import Response
```

```python
@app.get("/api/preview.jpg")
def preview_jpg():
    if not PREVIEW_PPM_PATH.exists():
        return JSONResponse({"error": "preview not ready"}, status_code=404)

    try:
        import cv2
        import numpy as np

        raw = np.frombuffer(PREVIEW_PPM_PATH.read_bytes(), dtype=np.uint8)
        img = cv2.imdecode(raw, cv2.IMREAD_COLOR)
        if img is None:
            return JSONResponse({"error": "preview decode failed"}, status_code=503)

        detect = _read_json_file(DETECT_JSON_PATH) or {}
        for box in detect.get("boxes", []):
            x, y, w, h = [int(v) for v in box]
            cv2.rectangle(img, (x, y), (x + w, y + h), (80, 220, 90), 3)

        ok, encoded = cv2.imencode(".jpg", img, [int(cv2.IMWRITE_JPEG_QUALITY), 70])
        if not ok:
            return JSONResponse({"error": "preview encode failed"}, status_code=503)
        return Response(content=encoded.tobytes(), media_type="image/jpeg")
    except Exception as exc:
        return JSONResponse({"error": str(exc)}, status_code=503)
```

Feasibility notes：

- `/dev/shm/preview.ppm` 是 raw PPM，通常是 5.76 MB 等級；每次轉 JPEG 會吃 CPU。
- Pi5 可以承受低頻手動預覽，但不應該讓手機背景 10 fps 一直拉。
- v2 建議：
  - counting tab active 才開始
  - default `2 fps`
  - 若使用者按「高更新率」才到 `5 fps`
  - tab hidden、screen lock、切回控制 tab 時停止

```javascript
function fmtNum(value, digits = 0) {
  const n = Number(value);
  if (!Number.isFinite(n)) return '—';
  return n.toFixed(digits);
}

async function refreshCount() {
  try {
    const r = await fetch('/api/count', { cache: 'no-store' });
    const payload = await r.json();
    const d = payload.data || {};
    const stale = payload.stale || !payload.ok;

    document.getElementById('count-total').textContent = d.total_crossed ?? '—';
    document.getElementById('count-live-tracks').textContent = d.live_tracks ?? '—';
    document.getElementById('count-fps').textContent = fmtNum(d.fps, 1);
    document.getElementById('count-throughput').textContent = fmtNum(d.throughput_per_min, 0);
    document.getElementById('count-speed').textContent = fmtNum(d.belt_speed_px_s, 0);

    const ageText = payload.age_s == null ? '尚無資料' : `${fmtNum(payload.age_s, 1)} 秒前更新`;
    const runText = d.paused ? '已暫停' : (d.running ? '採集中' : '待機');
    document.getElementById('count-subline').textContent =
      stale ? `計數狀態過期：${ageText}` : `${runText} · ${ageText} · batch ${d.batch_id || '—'}`;

    if (!stale && Number.isFinite(Number(d.throughput_per_min))) {
      countSamples.push({
        t: Date.now(),
        y: Number(d.throughput_per_min),
      });
      countSamples = countSamples.slice(-60);
      drawThroughputChart();
    }
  } catch (e) {
    document.getElementById('count-subline').textContent = '計數 API 連線失敗';
  }
}
```

```javascript
function drawThroughputChart() {
  const canvas = document.getElementById('throughput-chart');
  const ctx = canvas.getContext('2d');
  const w = canvas.width;
  const h = canvas.height;
  ctx.clearRect(0, 0, w, h);
  ctx.fillStyle = '#0f1117';
  ctx.fillRect(0, 0, w, h);

  ctx.strokeStyle = '#1e2130';
  ctx.lineWidth = 1;
  for (let i = 1; i < 4; i++) {
    const y = Math.round((h / 4) * i);
    ctx.beginPath();
    ctx.moveTo(0, y);
    ctx.lineTo(w, y);
    ctx.stroke();
  }

  if (countSamples.length < 2) return;

  const maxY = Math.max(10, ...countSamples.map(p => p.y));
  ctx.strokeStyle = '#66bb6a';
  ctx.lineWidth = 2;
  ctx.beginPath();
  countSamples.forEach((p, i) => {
    const x = countSamples.length === 1 ? w : i / (countSamples.length - 1) * w;
    const y = h - (p.y / maxY) * (h - 12) - 6;
    if (i === 0) ctx.moveTo(x, y);
    else ctx.lineTo(x, y);
  });
  ctx.stroke();

  ctx.fillStyle = '#78909c';
  ctx.font = '11px -apple-system, system-ui, sans-serif';
  ctx.fillText(`${Math.round(maxY)} /min`, 6, 14);
}
```

## 6. No disruption guarantee

計數核心不可被手機 UI 拖慢。設計約束如下：

- `preview_display.py` 每幀仍只做原本的 detector + tracker；新增 writer 最多每 `0.5s` 寫一次小 JSON。
- `count_status.json` 只含數字與狀態，不含 image、不含完整 frames、不含所有 tracks。
- 寫檔使用 `/dev/shm`，沒有 SD card I/O。
- 寫檔使用 atomic replace，FastAPI 讀壞 JSON 時不會讓 pygame 等待。
- `control_server.py` 不 import detector、不碰 Hailo、不呼叫 tracker，只讀檔。
- 手機端 polling `/api/count` 每 `1s`，`/api/batches` 每 `10s`，不反向呼叫 `preview_display.py`。
- v2 `/api/preview.jpg` 明確隔離為 on-demand，只有 counting tab 開啟才低頻 encode。

如果要更保守，可把 `COUNT_STATUS_WRITE_INTERVAL_S` 調成 `1.0`，手機仍有足夠即時感。

## 8. YAGNI and scope

### v1

Ship only:

- `/dev/shm/count_status.json`
- `GET /api/count`
- `GET /api/batches`
- phone cockpit counting tab
- live cumulative number
- live tracks
- fps
- throughput chart
- recent batches list

This gives the daily-use value immediately: the phone page controls capture and shows counting without SSH.

### v2

Add:

- `GET /api/preview.jpg`
- phone preview image polling at `2 fps`, only while counting tab is active

Reason for v2: PPM to JPEG encode is materially heavier than JSON polling and should be measured on Pi5 before making it always-on.

### 後續雲端

Plan C / proposal 12 remains the remote-beyond-Tailscale extension:

- Pi5 uploads batch summaries and live samples to Supabase.
- Cloudflare Pages hosts an authenticated cloud dashboard.
- It should reuse this plan's `count_status.json` contract as the local source of truth.

Do not mix cloud work into v1. The current plan is the Tailscale-local, single-phone-page integration.
