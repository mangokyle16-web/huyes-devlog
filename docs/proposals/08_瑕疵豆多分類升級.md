# 提案 08：瑕疵豆多分類升級 — 從「數豆子」到「分析品質」

> 由 Mac Mini Claude Code 在 Kyle 休息時自主思考產生（2026-06-10 深夜）

---

## 現況

目前 YOLO 只做 1 個類別（豆子 vs 非豆子）的偵測。  
多光譜相機已採集 720+ 筆資料，SAM2 labeling 工作流已成熟。

## 提案

將 label 從 1 類擴展成 5 類：

| 類別 | 說明 |
|------|------|
| 0: normal | 正常豆 |
| 1: black | 黑豆（烘焦/過熟）|
| 2: broken | 破裂豆 |
| 3: defect | 外觀瑕疵（蟲蛀/發霉）|
| 4: unripe | 未熟豆 |

## 為何現在是時機

- YOLO bbox 精度問題解決後，多分類是直接延伸
- Pi5 Hailo-8 支援單模型多類別，inference 時間幾乎不增加
- 現有 SAM2 labeling 流程可直接用於打多分類標籤

## 實作計劃

1. 重新 label 現有 dataset（單類 → 5 類），預估 2-3 小時
2. 用 `yolo-training-v3` 為基礎，新增 `yolo-defect.yaml` 重訓
3. 模型輸出從 `nc=1` → `nc=5`，後處理稍微調整

## 效果

系統從「計數」升級為「品質分析」，每顆豆子有分類標籤。
