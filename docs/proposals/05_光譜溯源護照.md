# 提案 05：光譜溯源護照 — 咖啡豆的數位身分證

> 由 Mac Mini Claude Code 在 Kyle 休息時自主思考產生（2026-06-06 凌晨）

---

## 一句話定義

每一批咖啡豆都有一份不可偽造的「光譜護照」——存入 Huyes 資料庫，讓任何人都能在 5 秒內驗證這批豆子是否原裝、是否摻假、是否來自聲稱的產地。

---

## 問題背景

台灣精品咖啡市場正快速成長，但也出現明顯的信任危機：

- **產地標示模糊**：部分業者將越南豆混入台灣豆，標榜「台灣精品」
- **烘焙度造假**：次級豆加深烘焙掩蓋瑕疵
- **摻假難以驗證**：目前業界沒有低成本的批次驗證工具

消費者、烘焙師、進口商都有驗真的需求，但現有方案（同位素分析、DNA 檢測）昂貴、耗時、需要專業實驗室。

**Huyes 的多光譜相機正好解決這個問題。**

---

## 核心概念：光譜即身分

每一批豆子都有獨一無二的「光譜簽名」，由以下因素決定：

| 因素 | 光譜特徵 |
|------|---------|
| 品種（Arabica/Geisha/...） | NIR 反射率模式 |
| 產地（台灣/衣索比亞/哥倫比亞） | 礦物質含量影響 NDRE |
| 處理法（水洗/日曬/蜜處理） | 糖分殘留影響 NDVI |
| 烘焙度 | 全波段系統性位移 |

這些特徵組合成 5 維的「光譜向量」，存入 Proposal 04 的光譜搜尋引擎（FAISS 資料庫）。

---

## 產品設計

### 護照的生命週期

```
農場/進口商 → 掃描生豆 → 產生護照 ID
     ↓
烘焙師 → 掃描前比對 → 確認真實性
     ↓
消費者 → 掃 QR Code → 查看完整溯源記錄
```

### 護照內容（一份 JSON）

```json
{
  "passport_id": "HYS-2026-TW-0001",
  "species": "Arabica",
  "origin": "台灣 阿里山 樂野",
  "process": "日曬",
  "harvest_year": 2025,
  "spectral_fingerprint": [0.61, 0.43, 0.18, 0.55, 0.29],
  "fingerprint_confidence": 0.97,
  "certified_by": "Huyes Taiwan Food Data Corp",
  "certified_at": "2026-03-15T08:30:00+08:00",
  "defect_rate": 0.02,
  "agtron_estimate": 63.4
}
```

### 驗證流程（對方有同型相機）

1. 烘焙師收到豆子，拿到輸送帶掃描
2. 系統比對光譜向量 vs 護照中的 fingerprint
3. 距離 < 閾值 → ✅ 真品，顯示溯源資訊
4. 距離 > 閾值 → ⚠️ 警告，建議退貨

### 驗證流程（對方沒有相機）

1. 農場/進口商開立護照，提供 QR Code
2. 消費者手機掃 QR Code → 開啟 Huyes 驗證頁
3. 頁面顯示：官方認證光譜圖、批次資訊、歷史掃描紀錄
4. 任何人都能看，但無法偽造（光譜向量存在 Huyes 伺服器）

## 商業模式

### 三層定價

| 層級 | 對象 | 內容 | 定價 |
|------|------|------|------|
| **免費** | 農場/小農 | 開立護照（每年 50 份） | NT$0 |
| **專業版** | 進口商/莊園 | 無限護照 + API 存取 + 批次驗證報告 | NT$3,000/月 |
| **企業版** | 拍賣平台/連鎖烘焙 | 白標 + 自訂域名 + SLA 保證 | 洽談 |

### 附加價值：Huyes 資料飛輪

每一份護照都讓光譜資料庫更豐富：
- 更多品種 → 辨識準確率更高
- 更多地區 → 地理 fingerprint 更精準
- 形成**護城河**：資料越多，競爭者越難複製

## 技術實作路徑

### Phase 1（4 週）：MVP 護照系統
- 擴充 `bean_spectra` SQLite 表，加入 `passport_id`、`origin`、`process` 欄位
- 建立 FastAPI endpoint：`POST /passports/create`、`GET /passports/{id}/verify`
- 簡單 Web UI：顯示護照資訊 + QR Code

### Phase 2（8 週）：驗證 API
- FAISS 比對引擎整合（Proposal 04）
- 多相機支援（允許不同型號相機的向量做跨校正）
- 護照簽章（用 Huyes 私鑰簽署，防偽造）

### Phase 3（12 週）：市場進入
- 與台灣咖啡協會合作試點
- 發布公開 API（讓其他驗證機構接入）
