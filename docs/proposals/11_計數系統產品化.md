# 提案 11：豆子計數系統產品化 — 從 demo 到分裝產品

> 由 Mac Mini Claude Code 在 Kyle 休息時自主思考產生（2026-06-14 凌晨）

---

## 背景

豆子計數系統已達 93%(370/399),實機運作。技術核心完成:
- fast-gray 快速灰階(fps 翻倍)
- YOLOX-tiny on Hailo-8
- 全域運動補償追蹤計數

下一步:把「計數 demo」變成「能賣的產品」。

## 五個方向

### A. 計數推到 98%+ — 提高相機出幀率
相機卡在 5.6fps(曝光允許更高,SDK 配置限制)。調查並解除 →
豆子每步移動 70-80px→35-40px → 計數 97-98%。

### B. 雙計數線交叉驗證
放兩條線(x=600/1000),漏偵測的豆子用另一條補回。純後處理,可疊加 A。

### C. 雲端看板 + 手機監控
Pi5 計數結果上雲(Supabase 免費層)→ 手機看今日總量/批次/流量曲線。
對接提案 06 手機遠端指揮台。

### D. 計數 → 重量估算 → 自動定量分裝 ★推薦
count × 平均單顆重 = 即時重量。設定目標重量達標 → GPIO 觸發停帶/閘門。
這把計數系統直接升級成**定量分裝機**,對接 optic_bean_sorter。

### E. 光譜瑕疵剔除
YOLOX bbox → 取原始 .qs 該位置光譜 → 判斷瑕疵 → 軌跡到末端 GPIO 吹除。
= Siamese 瑕疵偵測(提案 07)+ 計數 + 分豆機完整整合。

## 建議路徑

```
榨準確率: A + B → 97-98%
產品化:   D(定量分裝)最快變產品 → C(雲端賣點)
終極:     E(光譜瑕疵剔除)= 完整智慧分豆機
```

**推薦先 A+B(清技術債),再 D(定量分裝)** —— D 是最快變現產品的一步。
