## 你以為烘焙日期是什麼 vs 它實際上是什麼

**你以為：** 印在袋子上的「烘焙日期 6月14日」代表了新鮮度。
**實際：** 烘焙日期只是個時間戳記。同一天烘焙的兩批豆：

- 批次 A：真空鋁箔袋 + 氮氣充填 → 21天後光譜幾乎不變
- 批次 B：開蓋豆罐放室溫 → 7天後 VNIR B3-B7 段反射率已偏移

**日曆不知道差別。光譜相機知道。**

## 核心洞察：你已經有一台看得見氧化的儀器

咖啡豆烘焙後發生的化學變化，在 350–950 nm VNIR 範圍內是可偵測的：

| 化學過程 | 光譜影響 | 對應 Band |
|----------|----------|-----------|
| CO₂ 脫氣（初期，1–5天） | 表面孔隙率改變，漫反射增加 | B3–B5（470–650nm）|
| 脂肪氧化（中期，1–3週） | 脂質氧化產物吸收紅光 | B5–B7（590–770nm）|
| 美拉德產物降解（後期） | 可見區整體反射率下降 | B2–B6（全段）|
| 水分再吸附（存放條件差時） | 近紅外邊緣回應上升 | B9–B10（830–950nm）|

## 技術架構

### 三層設計

```
Layer 1: Spectral Baseline（烘焙當天）
  ├─ Pi5 輸送帶掃描整批豆（30–60秒）
  ├─ 記錄 10-band 平均光譜向量 V₀（已有 VNIR Phase 1 pipeline）
  ├─ 記錄 BQS 基線、Agtron 值（現有資產直接接）
  └─ 寫入 freshness_db：batch_id + t=0 row

Layer 2: Longitudinal Decay Tracking（T+3, +7, +14, +21, +28 天）
  ├─ 手機 cockpit 定時提示重掃（推播或 reminder card）
  ├─ 計算 spectral delta：ΔVₜ = Vₜ – V₀（每個 band 獨立）
  ├─ 計算 freshness_index：fi(t) = 100 × (1 – normalised_delta)
  └─ append 同一 batch_id 的新 time-point row

Layer 3: Model + Output
  ├─ 對每批豆的時序資料做指數衰退擬合（scipy.optimize，5行程式碼）
  ├─ 預測「fi 到達 75」和「fi 到達 45」的日期
  └─ 輸出 JSON freshness report → QR Code → 消費者手機
```

### 核心演算法（刻意保持簡單）

```python
import numpy as np
from scipy.optimize import curve_fit

def freshness_decay(t, fi0, k):
    """指數衰退 fi(t) = fi0 × exp(–k × t)"""
    return fi0 * np.exp(-k * t)

# 用前幾個時間點擬合
days      = [0,  3,  7, 14]
fi_values = [97, 91, 84, 71]

(fi0, k), _ = curve_fit(freshness_decay, days, fi_values, p0=[100, 0.02])

# 預測最佳窗口
t_bloom = int(np.log(fi0 / 88) / k)  # 排氣完成，甜味開始
t_peak  = int(np.log(fi0 / 70) / k)  # 最佳萃取截止
t_end   = int(np.log(fi0 / 45) / k)  # 建議轉奶咖或冷萃

print(f"最佳萃取窗口：第 {t_bloom} – {t_peak} 天，截止：第 {t_end} 天")
```

## 三個誠實的 Caveats

**1. Phase 0 可能失敗。**
若 VNIR 350–950nm 對咖啡氧化的靈敏度不足（全部 band 差異 < 1%），這個方向需要重新評估。但驗證只需要一個下午，代價極低，沒有任何硬體投入。

**2. 縱向資料收集需要紀律，不能衝刺完成。**
Phase 1 需要 2–3 週的定期重掃，而且每次掃描的環境條件（光線、溫度、相機位置）要盡量一致，否則噪音會掩蓋訊號。

**3. VNIR 只看得到部分氧化路徑。**
咖啡豆的芳香化合物揮發（讓你聞到香氣的那些分子）在 350–950nm 大多不直接可見；本提案測量的是**表面光學特性的變化**，作為新鮮度的代理指標（proxy）。對消費者用途夠用，但不等於全面的化學分析。
